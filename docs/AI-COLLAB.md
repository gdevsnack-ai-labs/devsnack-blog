# AI-COLLAB: AI와 함께하는 블로그 관리

> 이 프로젝트는 **Hermes Agent (AI)**와 **사장님 (사람, 문규 강)**의 협업으로 운영됩니다.
> 모든 과정 — 설계, 코딩, 배포, 장애 대응 — 이 기록으로 남아 있습니다.

---

## 🤝 협업 방식

| 역할 | 누가 | 하는 일 |
|:-----|:----|:--------|
| **방향 결정** | 사람 | "이렇게 해볼래?", "이상한데?", "ㄱㄱ" |
| **초안 작성** | AI | 코드 구현, 문서 초안, 설계안 |
| **검증** | 사람 | 결과 확인, 문제 발견, 피드백 |
| **수정** | AI | 피드백 반영, 재시도, 최적화 |
| **기록** | AI | 위키 정리, git commit, 로그 보관 |

## 📋 실제 예: 2026-07-20 리디자인

```
1. 사람: "Vercel 위키 읽어보고, 프로젝트 페이지 설계해볼래?"
   AI:  위키 분석 → GPT 의견 참고 → 설계안 제안

2. 사람: "마음에 든다. 하나씩 계획 문서 만들고 진행하자"
   AI:  docs/PLAN-2026-07-20.md 작성

3. AI:  Phase 1~5 순차 구현 (데이터 모델 → Lab 페이지 → LNB → 모바일)
   AI:  각 Phase 빌드 검증 → 스크린샷 보관

4. 사람: "최종 버전만 확인할게"
   AI:  로컬 커밋 후 완료 보고
```

## 💡 철학

1. **"Pull, not Push"** — AI가 credential을 원격에 올리지 않고, 로컬에서 정제해서 원격이 가져가는 구조
2. **"발전하는 블로그"** — 미완성도 공개, 진행률 표시, 진화 과정을 그대로 노출
3. **"Lab = 콘텐츠 허브"** — 하나의 프로젝트가 블로그 시리즈 + YouTube + GitHub로 연결
4. **"실패도 기록"** — 장애, 디버깅, 잘못된 결정도 모두 로그로 남김
